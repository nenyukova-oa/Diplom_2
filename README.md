# Diplom_2
## Задание 2: Автотесты для API

### Автотесты для проверки программы, которая помогает заказать бургер в Stellar Burgers

### Реализованные сценарии

Созданы следующие автотесты для API:

Создание пользователя:
- создать уникального пользователя;
- создать пользователя, который уже зарегистрирован;
- создать пользователя и не заполнить одно из обязательных полей.

Логин пользователя:
- вход под существующим пользователем;
- вход с неверным логином и паролем.

Создание заказа:
- с авторизацией;
- без авторизации;
- с ингредиентами;
- без ингредиентов;
- с неверным хешем ингредиентов.


### Структура проекта

`tests` - пакет, содержащий тесты:
- test_create_user - создание пользователя
- test_login_user - логин пользователя
- test_order - заказы
`api_methods` - пакет с методами для тестов:
- create_user - создание пользователя
- login_user - логин пользователя
- order - заказы

### Запуск автотестов

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск автотестов и создание отчета в аллюр**

>  `$ pytest tests --alluredir=allure_results`
